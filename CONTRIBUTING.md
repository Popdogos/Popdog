# Contributing to Popdog

Thank you for your interest in contributing to Popdog! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on what is best for the community
- Show empathy towards other community members

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:
- A clear title and description
- Steps to reproduce the bug
- Expected vs actual behavior
- Browser and OS information
- Screenshots if applicable

### Suggesting Features

Feature suggestions are welcome! Please create an issue with:
- A clear description of the feature
- Use cases and examples
- Potential implementation approach (if you have ideas)

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add feature: description'`)
6. Push to your branch (`git push origin feature/your-feature-name`)
7. Open a Pull Request

### Coding Standards

- Use semantic HTML5
- Follow existing CSS naming conventions
- Write clean, readable JavaScript
- Comment complex logic
- Ensure mobile responsiveness
- Test in multiple browsers

### Commit Messages

Use clear, descriptive commit messages:
- Start with a verb (Add, Fix, Update, Remove)
- Be specific about what changed
- Keep messages concise but informative

Example:
```
Add mobile optimization for Send PopDog section
Fix wallet connection error handling
Update README with installation instructions
```

## Development Setup

1. Clone the repository
2. Open `index.html` in a local server
3. Make your changes
4. Test in multiple browsers and devices
5. Submit your PR

## Questions?

Feel free to open an issue for any questions or reach out to the community!
